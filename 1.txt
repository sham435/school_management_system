### Root Cause Analysis

**Primary Issue Identified:**
- Frontend uses `tenantId = 'default-tenant-id'` (hardcoded string in page.tsx line 41)
- Database schema expects UUID format (`@db.Uuid` for tenantId in QAEvaluation model)
- When Prisma queries with this invalid UUID format, it causes 500 Internal Server Error

**Secondary Issue (Hydration Warning):**
- Using `toLocaleDateString()` which is locale-dependent
- Causes mismatch between server-rendered HTML and client hydration

### Proposed Plan (Shared with User):
1. Fix 500 error - add error handling in backend
2. Fix hydration mismatch - add mounted state in frontend
3. Test the fixes

#### Change 3: Fixed date rendering (line 288)
**Before:**
```typescript
{eval_.createdAt ? new Date(eval_.createdAt).toLocaleDateString() : '-'}
```

**After:**
```typescript
{mounted && eval_.createdAt ? new Date(eval_.createdAt).toISOString().split('T')[0] : '-'}
```

**Reason:** `toLocaleDateString()` returns different strings based on user locale, causing hydration mismatch. Using ISO date string is consistent between server and client.
